Design: Junction Graph + Path Planning

Section 3 of 4 — the core of what makes the line feel artistic rather than mechanical.

Step 1: Build the junction graph from skeleton

After Zhang-Suen thinning, every edge pixel is exactly 1px wide. We classify each lit pixel by how many lit neighbors it has:

Endpoint
exactly 1 neighbor — a tip, a line end
Path pixel
exactly 2 neighbors — interior of a branch
Junction
3+ neighbors — a fork, a crossing

From endpoints and junctions, trace branches: walk along path pixels until you hit the next endpoint or junction. Each branch stores its pixel sequence (for smooth spline fitting later) and its length. Prune branches shorter than 0.02 × d.

Step 2: Weight branches by importance

Not all edges are equal. We want the silhouette to dominate. Each branch gets a weight:

weight = branchLength × (1 + silhouetteBonus × isSilhouette)

isSilhouette = 1 if this branch borders the image's background region (white/transparent pixels adjacent to edge pixels). silhouetteBonus defaults to 2.0 — silhouette edges are treated as 3× as important as interior edges. This is the "C insight" baked into approach B.

Step 3: DFS spanning tree → Euler tour

We want one continuous path through all branches. The strategy:

  1. Build a maximum spanning tree of the graph (Kruskal's, using branch weights). This selects the most important subset of branches that keeps everything connected.
  2. A tree has all nodes at odd or even degree. For an Euler tour, every node must have even degree. Add duplicate (ghost) copies of the minimum-weight edges needed to fix odd-degree nodes — this is the Chinese Postman trick. Each ghost edge will be traversed as a smooth connector, not a visible retrace.
  3. Run Hierholzer's algorithm to extract a single Euler circuit. At each junction, prefer unvisited real edges over ghost edges, and among real edges prefer the longest branch — this keeps the line flowing along structure rather than darting around.

Result: one closed loop that visits every important branch exactly once, with short ghost connectors linking disconnected regions. Ghost connectors become the invisible pen-up moves (or very short drawn jumps) only when the distance exceeds maxJump × d.

Step 4: Catmull-Rom spline fitting

The Euler tour gives an ordered list of skeleton pixels. We don't draw pixel-by-pixel — we subsample the path at every ~0.005 × d pixels (keeping only shape-defining points), then fit a Catmull-Rom spline through them. Catmull-Rom passes through its control points (unlike Chaikin which shrinks them), producing curves that faithfully follow the skeleton without over-smoothing sharp corners.

The spline is converted to cubic Bézier segments for SVG — compatible with the existing SVG exporter.
