Design: Size-Agnostic Parameters

Section 2 of 4 — all pixel-based thresholds expressed as fractions of image diagonal.

The problem

All current parameters are in absolute pixels. A 600px image and a 300px image get the same minContourLen: 20, so the 300px image loses twice as much detail. The fix: express everything as a fraction of the image diagonal d = √(W² + H²). The diagonal is scale-invariant and captures both dimensions.

Parameter table

Parameter Old (absolute px) New (fraction of d) Effect
blurSigma radius = 1 0.006 × d 600px image → σ≈5px. Controls texture suppression before edge detection.
cannyHigh threshold = fixed 0.15 (of max grad) Relative to gradient magnitude max — already scale-invariant.
cannyLow — 0.05 (of max grad) Hysteresis lower bound. Weak edges survive only if touching a strong one.
minBranchLen minContourLen = 20 0.02 × d 600px → 16px. Prunes tiny skeleton branches that would create noise in the path.
maxJump maxJumpDist = 60 0.08 × d 600px → 64px. Max pen-up distance before we insert a null gap instead of drawing.
splineTension smoothIterations = 3 0.0–1.0 (unitless) Catmull-Rom tension. 0 = loose/curvy, 1 = follows points tightly. Default 0.5.

MAX_SIZE stays, but params scale with it

The 600px cap is useful for performance. We keep it. The point is that if you change it to 1200px (or remove it entirely), all the parameters automatically produce the same visual effect — because they're fractions of the diagonal, not hardcoded pixel counts. The UI sliders will show the fraction values (e.g. "0.02") with a live px preview for the current image.

Slider UX change

Current sliders show raw pixel values. New sliders show the fraction + a parenthetical pixel equivalent computed from the current image's diagonal. Example: Min Branch 0.020 (≈ 16 px). This makes tuning intuitive regardless of input size.
