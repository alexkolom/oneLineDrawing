Three approaches to improve the pipeline

Each trades implementation effort against output quality. Pick the one that feels right.

A

Evolutionary — patch the current pipeline

Keep the posterize→regionBoundaries flow. Fix the worst pain points: normalize all parameters by image diagonal so they're size-agnostic; improve junction handling in ContourTracer so it makes smarter choices at forks; add a depth-first spanning tree traversal to replace the chaotic greedy connector.

Pros

  • Low risk — preserves what works
  • Fastest to ship

Cons

  • regionBoundaries produces blocky grid-like edges, not organic contours — this is the root cause of "looks graphical not artistic" and can't be fully fixed by patching downstream
B

Architectural rewrite — Canny → Skeleton → Graph → DFS path RECOMMENDED

Follow the academic pipeline: replace edge detection with Canny (nonMaxSuppression already exists in EdgeDetector.js); add Zhang-Suen skeletonization to thin edges to 1-pixel centerlines; build a proper junction graph (nodes = endpoints + junctions, edges = skeleton branches); plan the path with a DFS spanning tree biased toward long/prominent branches; fit B-spline through the result. All parameters expressed as fractions of image diagonal.

Pros

  • Produces organic, flowing lines — addresses the root cause
  • Junction graph enables artistically intelligent path decisions
  • Naturally size-agnostic
  • Most closely matches how artists actually draw

Cons

  • Requires new Skeletonizer and JunctionGraph modules (~3–4 new files)
  • More moving parts to tune
C

Silhouette-first — exploit the clean background

Since the background is removed, extract the outer contour explicitly (largest connected boundary). Then find 2–3 tiers of interior structural edges (by gradient strength). Route a single loop: start on the silhouette, spiral inward through interior features, return to the silhouette. Jumps only when interior features are genuinely disconnected islands.

Pros

  • Perfectly matched to the Minimal Silhouette aesthetic
  • Silhouette always gets priority — the subject is never lost
  • Simpler than full graph rewrite

Cons

  • Brittle — breaks badly if background isn't cleanly removed
  • Interior routing is still heuristic, not graph-principled

My take

B is the right architectural move — it fixes the root cause and builds infrastructure that makes further artistic tuning tractable. C's silhouette insight can be layered on top of B: use the background mask to weight the silhouette edges highest when building the DFS spanning tree. Best of both.
